# Provenance Pack

The **Provenance Pack** is the third core primitive in the Pictovap pipeline.
It is a persistent audit trail for every image that passes the `FitScore` gate
and is selected for placement.

Pictovap does not generate images and does not guarantee legal compliance.
The Provenance Pack is a structured record that makes the origin, license, and
processing history of each selected image traceable and inspectable.

## What It Records

For every selected image, a `ProvenancePack` stores:

| Field | Description |
|---|---|
| `image_id` | Unique identifier for this image in the pipeline |
| `source_type` | `local`, `api`, or `url` |
| `provider` | Source adapter name (e.g., `local`, `unsplash`, `depositphotos`) |
| `source_url` | Original URL if fetched from an API |
| `local_source_path` | Filesystem path if sourced locally |
| `license_status` | License string (e.g., `CC0`, `owned`, `unsplash`, `editorial`) |
| `attribution` | Required attribution string if any |
| `original_filename` | Filename as received from the source |
| `generated_filename` | Final output filename after processing |
| `content_hash` | SHA-256 hash of the original file content |
| `article_id` | The article this image was selected for |
| `slot_id` | The image slot within the article |
| `placement_target` | The CMS injection target (e.g., `featured_image`) |
| `generated_alt_text` | Alt text generated for accessibility and SEO |
| `generated_caption` | Caption generated for display |
| `timestamp` | UTC timestamp when the pack was created |
| `processing_actions` | List of applied transformations (e.g., `resize_1200`, `webp_convert`) |

## What Generates Metadata

Alt text and captions are generated by whatever metadata adapter is configured.
In production deployments, this may be Gemini, Claude CLI, or LM Studio.
In the local demo, they are generated from keyword templates — no AI API required.

The `ProvenancePack` stores the result regardless of which generator produced it.
It does not depend on any specific AI provider.

## Example

```python
from pictova.core.primitives import ProvenancePack

pack = ProvenancePack(
    image_id="img-backpack-01",
    provider="local",
    source_type="local",
    local_source_path="examples/assets/minimal-backpack.jpg",
    license_status="CC0",
    original_filename="minimal-backpack.jpg",
    generated_filename="pictovap_minimal-backpack.webp",
    content_hash="66007ff9d92b261c",
    article_id="demo-article-001",
    slot_id="featured",
    placement_target="featured_image",
    generated_alt_text="A backpack, travel, minimalist scene",
    generated_caption="Visual for: The Future of Minimalist Travel",
    processing_actions=["resize_1200", "webp_convert", "exif_strip"],
)

print(pack.to_dict())
```

## Where It Feeds

The `ProvenancePack` passes into the `CMSPlacement` stage, which reads it to
construct the final placement instructions, including alt text and captions.
