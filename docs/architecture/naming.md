# Brand & Naming Doctrine

## The Meridyen Ecosystem

**Meridyen** is the umbrella platform behind [yoldaolmak.com](https://yoldaolmak.com) — a travel content platform built entirely with purpose-built, AI-powered tools. Each tool in the ecosystem is an independent application that can stand alone commercially, but was born to serve Meridyen.

```
Meridyen (platform)
├── YOOS-APP   → content layer (article generation engine)
└── Pictova    → visual layer  (image intelligence engine)
```

The naming pattern follows a deliberate philosophy: every tool gets a coined name that is globally unique, memorable across languages, and unambiguous as a brand.

## Why Coined Names

Generic names like "ImageAI" or "VisualPicker" are forgettable, un-ownable, and impossible to search. The Meridyen ecosystem uses the same strategy Graphova pioneered:

- Take a root with clear semantic meaning
- Add a suffix that signals novelty, momentum, or transformation
- Arrive at something that sounds like a proper noun, not a feature

## Graphova and Its Descendants

**Graphova** was the first application built on this pattern.

| Name | Root | Meaning | Layer |
|------|------|---------|-------|
| Graphova | *graphein* (Gr: to write) | Writing intelligence | Main SaaS |
| Penova | *pena* (pen) | Travel writing layer | Content prototype |
| Penovate | *pena* + *innovate* | Fact-freshness & QA | Quality layer |
| Evowrite | *evolve* + *write* | Evolving content | Growth layer |
| Tonova | *tone* | Voice & tone variants | Style layer |
| Merchova | *merch* | Commercial content | Monetization layer |

## Pictova

**Pictova** follows the same grammar.

| Component | Value |
|-----------|-------|
| Root | *pictus* (Lat: painted, depicted) |
| Suffix | *-ova* (nova = new; signal of momentum) |
| Full read | "depicted nova" — the visual layer, renewed |
| CLI | `pictova` |
| Package | `src.pictova` |
| Tagline | *Visual Intelligence for Content* |

### Why *pict-* and not *photo-*, *imag-*, or *vis-*?

- **photo-**: overused in product names, no differentiation
- **imag-**: generic, easily confused with "imaginary"
- **vis-**: sounds like an internal codename
- **pict-**: Latin *pictus* (past participle of *pingere*, to paint) — carries the act of depiction, not just the noun "picture". It is precise, uncommon in product naming, and pairs cleanly with *-ova*.

## Sub-layer Names (Planned)

As Pictova matures into a standalone commercial product, sub-layers follow the same root:

| Sub-layer | Purpose |
|-----------|---------|
| **Pictova Select** | Semantic selection engine |
| **Pictova Depot** | DepositPhotos and licensed stock integration |
| **Pictova Memory** | Visual memory index (Mac Photos + local) |
| **Pictova Frame** | Image formatting and block placement |

## Naming Rules

Any future sub-layer or variant must satisfy:

1. Globally unique — searchable without disambiguation
2. Pronounceable in English and Turkish
3. No trademarked terms, no generic English adjectives
4. Derived from the same root (*pict-*) or the *-ova*/*-ate* pattern
5. Stands alone as a proper noun

## Pictova vs Graphova: Positioning

| | Graphova | Pictova |
|--|----------|---------|
| Domain | Written content | Visual content |
| Input | Post briefs, outlines | Post context, image sources |
| Output | Articles, drafts | Placed images with metadata |
| Primary surface | Web dashboard + API | CLI + HTTP API |
| Born for | Any content publisher | WordPress / Meridyen first |
| Ambition | Any writer × any content | Any publisher × any image source |
