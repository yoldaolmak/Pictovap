# Pictovap v0.2.0 Release Notes

This release establishes the public open-source core: credential-free demo, documented primitives, adapter model, tests, and contribution paths.

## What is Pictovap?

Pictovap is an open-source visual finishing engine for content publishers. It turns unfinished articles into visually complete, rights-aware, publish-ready CMS pages.

## What Works in This Release

The core pipeline is fully operational locally without requiring any credentials or external APIs. You can run the deterministic test suite and a credential-free local demo.

The pipeline executes through four core primitives:
1. **Visual Brief**: Analyzes article structure to generate image requirements.
2. **Fit Score**: Evaluates candidates against contextual, technical, and licensing criteria.
3. **Provenance Pack**: Maintains an audit trail for selected images.
4. **CMS Placement**: Produces CMS-agnostic placement instructions.

## How to Run the Demo

You can run the built-in credential-free demo using:

```bash
make demo
```

Or using the Python module directly with your own article:

```bash
python -m pictova.demo --article examples/sample-article.md --profile examples/profiles/sample-publisher.yaml --output examples/sample-output.json
```

## What the Demo Does Not Do

- It does not make live calls to external model providers or paid APIs.
- It does not fetch live stock photos from Unsplash or DepositPhotos.
- It does not push actual content to a live CMS like WordPress.
- It relies entirely on mock assets and deterministic scoring to demonstrate the pipeline architecture reliably.

## Current Adapter Status

- **Image Sources**: Includes references for Unsplash, DepositPhotos, and a local visual memory database.
- **CMS Adapters**: The WordPress adapter is included as a reference implementation extracted from dogfooding usage.
- All adapters are considered reference implementations at this stage. 

## Current Limitations

- There is no external adoption yet. The system was extracted from dogfooding at a single publisher.
- The pipeline logic is still maturing and decoupling from legacy dictionary passing into strict typed primitives.
- The default setup only handles English or Turkish structural extraction reliably.

## Known Risks

- **Provenance != Legal Guarantee**: The Provenance Pack provides an audit trail of image sources and licenses, but it is not a legal guarantee. Publishers remain responsible for ensuring correct licensing.
- **Internal Namespace**: The internal Python package namespace remains `pictova` for backward compatibility, while the product name is `Pictovap`.
- **WordPress State Drift**: Using the WordPress adapter without maintaining persistent state for `PICTOVA_POST_MANIFEST_DIR` across deployments could lead to orphaned media blocks in the CMS.

## Contribution Areas

We welcome contributions! Please see our [Starter Issues](../contributing/starter-issues.md) backlog. Key areas of interest include:
- Adding new image source adapters (e.g., Openverse).
- Adding new CMS placement adapters (e.g., Ghost, Strapi).
- Improving alt text generation templates.
- Expanding local folder image loading capabilities.
