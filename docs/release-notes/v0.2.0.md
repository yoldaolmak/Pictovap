# Pictovap v0.2.0

This marks the first public Open Source release of Pictovap, an early-stage infrastructure project for content publishers. (Internal prototyping phases were versioned as 0.1.x).

Pictovap is a headless, modular pipeline that transforms raw text content into a fully visually-populated publishing package. It addresses the gap between writing an article and placing images into a CMS.

## What Works

* **Core Pipeline**: A sequential flow from Analysis → Selection → Processing → Placement.
* **Core Primitives**: The pipeline passes typed `VisualBrief`, `FitScore`, `ProvenancePack`, and `CMSPlacement` objects.
* **Local Demo**: You can run the entire pipeline locally without any credentials or API keys.
* **Custom Article Parsing**: The demo accepts custom Markdown files to generate specific visual briefs.
* **Adapter Architecture**: Documented extension points for image sources and CMS placement.
* **WordPress Adapter**: A working reference implementation for pushing to WordPress.

## How to Run the Demo

Get started immediately with the credential-free mock path:

```bash
git clone https://github.com/yoldaolmak/Pictovap.git
cd Pictovap
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
pip install -e .

make demo
```

To test it against your own article:
```bash
python -m pictova.demo --article path/to/your/article.md --output my-plan.json
```

## Adapter Status

* **CMS Adapters**: WordPress (Working), Ghost (Stub), Strapi (Stub).
* **Source Adapters**: Local files, Unsplash (Mock/Stub).

## Known Limitations

* Core logic was recently extracted from production; some internal dictionaries are still being migrated to strict dataclass primitives.
* The local demo uses mock image assets and deterministic scoring rather than live API calls.
* Publisher profiles exist as Python dataclasses but are not yet fully loadable from YAML in all pipeline paths.
